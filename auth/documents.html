Project Setup Guide

1. Create Virtual Environment

python -m venv env

2. Activate Environment

Windows:

.\env\Scripts\activate

macOS/Linux:

source env/bin/activate

3. Install Dependencies

pip install -r requirements.txt

4. Run Development Server

python manage.py runserver 4444
Note: Make sure you have created a .env file with all required environment variables before starting the server.

Project Features

  • User login & registration system
  • Custom admin and editor dashboard authentication
  • Password recovery functionality
  • Environment variables management using python-decouple
  • Role-based access control (Admin, Editor, Customer)
  • Secure password handling

Environment Variables

The project uses the following environment variables (set in .env file):

Required Variables

  • SECRET_KEY - Django secret key
  • DEBUG - Debug mode (True/False)
  • ALLOWED_HOSTS - Comma-separated list of allowed hosts
  • DATABASE_URL - Database connection URL

Email Configuration

  • EMAIL_HOST - Email server host
  • EMAIL_PORT - Email server port
  • EMAIL_HOST_USER - Email username
  • EMAIL_HOST_PASSWORD - Email password
  • EMAIL_USE_TLS - Use TLS (True/False)
Security Note: Never commit your .env file to version control. Add it to your .gitignore file.
